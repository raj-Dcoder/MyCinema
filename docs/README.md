# MyCinema Docs

Start here when deciding which project process to follow.

## Development And Release Flow

- `DELIVERY_WORKFLOW.md` - use this when the user reports a problem, bug, issue, or feature. It covers intake, analysis, branch setup, implementation, verification, user acceptance, and the handoff to release.
- `release/00-release-map.md` - use this after the user confirms there are no more implementation items and wants to move toward deployment or release. A normal handoff from the delivery workflow starts at release gate `04-release-record.md`.

## Release Support

- `WINDOWS_CODE_SIGNING.md` - Windows signing setup required for public signed installer releases.
- `release/*.md` - numbered release gates for versioning, release notes, What's New, packaging, publishing, and post-release verification.
