# 15 Package And Publish

Use this gate only after the release commit has been pushed and the user has approved publishing.

The goal is to package the app and publish the GitHub release.

## Preconditions

Before running publish, confirm:

1. `npm run build` passed in `11-local-release-verification.md`.
2. The complete version changeset was committed and pushed in `13-release-commit-and-push.md`.
3. `GH_TOKEN` was loaded from the Windows User environment in `14-github-token.md`.
4. Code-signing credentials are available in the build environment.
5. The approved version is still consistent across release files.

## Publish Command

```powershell
npm run release:publish
```

Wait for the command to finish. This can take several minutes.

Then verify the generated Windows signatures:

```powershell
npm run verify:installer-signature
```

## Success Requirements

1. packaging completes
2. installer/artifacts are generated
3. Windows installer and app executable signatures are valid
4. upload to GitHub succeeds
5. GitHub release uses tag/title `vX.Y.Z` or the version format generated by electron-builder for this project

## Failure Handling

If publishing fails:

1. report the exact failing stage
2. include the key error message
3. do not claim the release was published
4. do not continue to published release verification
